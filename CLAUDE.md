# あみわり（amiwari）開発メモ（Claude Code用）

編み物の「割り出し計算」PWA。**マネタイズ前提**（無料+買い切り解放、乙4ドリルと同じ解除コードモデル）。
テーマ候補「編み物・漢方・相撲」から編み物を選定（2026-07-10）。
漢方は薬機法リスクと製薬会社の無料体質診断が競合するため、相撲はデータ権利と課金動機の薄さで見送り。

## 企画（2026-07-10 起案。ユーザーレビュー待ち）

- **ターゲット**: 棒針編みの中級者。ウェア・小物を「自分のゲージ・自分のサイズ」で編みたい人（30〜60代女性中心）
- **課題**: 自分のゲージがレシピと違うと目数・段数を全部再計算する「割り出し」が必要。
  分散増減目（「◯段ごとに◯目減らす」）の平均計算は手計算だとミスしやすく、本来は編み物教室で習う専門技能
- **競合と隙間**: 定番の無料アプリ ハマナカ「ニット電卓」は**セットインスリーブ・ラウンドネックのプルオーバー専用**。
  ラグラン・Vネック・帽子・靴下は非対応で、ユーザー要望が出たまま長年拡張されていない。
  他は単機能のWeb電卓（ゲージ換算のみ）とブログの手計算解説だけ → **「セーター以外・ラグラン」の割り出しツールが空白**
- **差別化**: ニット電卓が対応しない形状を最初から狙う。着手順は 帽子 → 靴下 → ラグランセーター
  （帽子が最も計算が単純で検証しやすく、編む人口も多い）
- **マネタイズ**:
  - 無料: ゲージ換算・平均計算（分散増減目）・段数カウンター → 「ラグラン 割り出し 計算」等の検索流入の受け皿
  - 買い切り: 形状別割り出し（寸法+ゲージ入力 → 目数段数表と編み方テキストを出力）
  - 決済・解除コード方式は乙4と同じ（SHA-256ローカル照合）。決済アカウントは未契約（乙4と共通の課題）
- **価格仮説**: 600〜980円。編み物層は毛糸・道具・有料編み図（minne等で1枚300〜800円）に日常的に払っており、
  資格ドリル層より支払い抵抗が低いと仮定

## 品質ルール（最重要）

- 割り出しは**計算を間違えたら商品として致命的**（乙4の誤答と同じ）。ロジックは編み物の定石「平均計算」に基づき、
  既知の作例と突き合わせるテストケースを必ず用意する
- 計算結果の実地検証（試し編み）はアプリ内だけでは完結しない。公開前にユーザーまたは編み物経験者の試用を挟む
- 用語は日本の編み物本の標準表記（目・段・ゲージ・伏せ目など）に合わせ、独自用語を作らない

## デザイン方針(2026-07-12 ユーザー提供の参考資料をもとに再刷新)

- **「一目でClaude作とわかるデザイン」は禁止**。過去2回の経緯:
  初版(クリーム+テラコッタ+丸角カード+絵文字)→NG。
  2版(藍+からし手芸ノート風・実線枠+ずらし単色影)→「ネオブルータリズム風もAI定番」でNG。
  均質さ・破綻のなさ自体がAIっぽさの正体、という診断で3版へ
- 現在は**ユーザー所有の手芸書のキャプチャ(`参考資料/`。書籍紙面なので絶対にコミット禁止)を引き写した
  「現代の手芸書」風**:
  - 淡いピンクの紙(#f5dfdb)に、白の有機的な丸パネル(枠線・影なし。角丸は4隅バラバラの大きめ半径)
  - 見出し・ボタン・数字: Zen Maru Gothic(丸ゴ)のティール(#4cb1a3 / 濃 #2e9184)
  - 本文・説明文: Shippori Mincho(明朝)の小さめ・行間広め
  - 各ツールに英字小ラベル「Tool 01」(参考書の「Point 01」形式)。アイコンタイルとチェブロンは廃止
  - ボタンは丸ピル(border-radius:999px)。結果表示は淡ティールの方眼(編み図用紙)
  - ホーム冒頭は雲形パネル+毛糸玉線画(参考書のNavigator枠の引き写し)
- **コピーは「一人の編み手の声」で不均一に**: 全項目を同じ丁寧さで説明しない。
  平均計算は熱く、カウンターは素っ気なく(「数えるだけ。閉じても消えないだけ。」)。
  マーケ文句と「— 〜 —」のダッシュ囲いタグラインは禁止
- 注記は「※」を使う。絵文字は原則使わない

## 技術方針

- HTML + CSS + Vanilla JS、依存なし・ビルドなし（既存プロジェクトと同じ）
- PWA（ホーム画面追加・オフライン動作）。スマホ縦画面ファースト（編みながら片手で見る想定）
- カウンターは画面ロック中も押しやすい大ボタン・誤タップ取り消し付き

## ファイル構成

- `index.html` — 全画面のHTML + CSS（見た目はすべてここ）
- `js/calc.js` — **計算ロジック（純粋関数のみ）**。UIから独立させテスト対象にしている
- `js/main.js` — 画面遷移・入力バインド・カウンター保存・解除コード照合
- `tests/calc.test.mjs` — 計算ロジックのテスト（作例との突き合わせ + 性質チェックの総当たり）
- `tests/preview.html` — スマホ幅検証用ラッパー（下記「検証手順」参照）
- `tests/counter-test.html` — 段数カウンターの動作検証ハーネス（旧データ引き継ぎ・追加・切替・削除・永続化）。
  ヘッドレスChromeで `--dump-dom` して `<pre>` のログを見る
- `manifest.json` / `sw.js` — PWA（swはネットワーク優先。push後の更新は普通に反映される）
- `favicon.svg` / `favicon.png` — ファビコン（藍地に毛糸玉の線画。pngはicon-512から `sips -z 48 48` で生成）
- `icon-source.html` — PWAアイコンの元（毛糸玉SVG。favicon.svg・ヘッダーロゴと同じパス）。
  再生成: ヘッドレスChromeで512pxスクショ → sipsで192/48に縮小

## 機能（現状）

- 📐 ゲージ計算（無料）: 寸法→目数・段数 / レシピのゲージ→自分のゲージ換算
- ➗ 増減目の平均計算（無料）: 「◯段ごとに1目×◯回」（袖下など）と「◯目編んで2目一度×◯回」（1段内の分散）。編み図表記（段-目-回）も併記
- 🔢 段数カウンター（無料）: 段数 + 模様リピートの2連。目標段数で残り表示。
  **「編みかけ」単位で複数保存**（チップで切替、名前変更・削除可。localStorageキー `amiwari-counter-v2`。
  旧キー `amiwari-counter` は初回に1件目へ自動引き継ぎして削除）
- 🧢 帽子の割り出し（本来🔒買い切り。**現在は無料β公開中**）: 頭囲・深さ・ゆとり・ゴム編み・ゲージ・分割数 → 手順書を生成
  - βは js/main.js の `HAT_BETA = true` で制御。βカード（帽子画面上部）とホームの「無料β」バッジも
    このフラグで出し分け。**β終了時は false に戻し解除コードも差し替える**
  - 感想募集リンクは js/main.js の `FEEDBACK_URL`（Googleフォーム）。**空のあいだはリンク非表示**。
    ユーザーがフォームを作成したらURLを設定する（メールアドレス公開はNGのため mailto は使わない）
  - クラウンは「2段ごと→毎段」の定番減目構成。作り目はゴム編みと分割数の公倍数に自動で丸める
- ホーム画面に追加（A2HS）動線: ホーム下部のカード。Android/Chrome系は beforeinstallprompt で
  本物のインストール、iOS Safariは共有メニューの手順ガイドを開閉。standalone起動時は非表示。
  `?install=demo` で強制表示（スクショ検証用）
- 解除コード: `AMIWARI-HAJIME-2026`（SHA-256照合。ハッシュは js/main.js の UNLOCK_HASH）。
  **販売開始時に必ず新コードへ差し替えること**

## 変更後の検証手順

1. `node --check js/main.js && node --check js/calc.js`
2. **`node tests/calc.test.mjs`（計算を触ったら必須。全部通ること）**
3. ローカル確認: `python3 -m http.server 8789`（ES modulesなので file:// 不可）
4. スマホ幅スクショ: macOSのChromeは最小幅500pxなので **`--window-size=500,920` で
   `http://localhost:8789/tests/preview.html?scr=hat&unlock=1` を撮る**。
   `scr=`（画面）/ `pane=`（タブ）/ `set-<inputのid>=値` / `unlock=1` / `h=1450`（iframe高さ）が使える
5. index.html 自体も `?scr=gauge` のディープリンクで直接画面を開ける
